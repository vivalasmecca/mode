# Brand Brief

## Tone of voice
Direct, precise, quietly confident. Assumes the reader is technical 
and smart — never over-explains. Provocative enough to create a 
pause, grounded enough to be trusted. No hype language. No 
adjective stacking. The product makes a strong claim and then 
proves it.

## Messaging pillars
1. Intelligence, not configuration — MODE makes decisions the 
   design system couldn't make before. Not more settings, more 
   judgment.
2. Code-first, not tool-first — lives in your codebase, runs in 
   your pipeline, owned by your team.
3. The layer that was missing — not a replacement for your design 
   system, the semantic layer above it that makes it aware.
4. Built for the moment AI made design systems dumb — everyone 
   can build now, which means coherence is the new craft.

## Positioning statement
MODE is the intelligence layer for code-first SaaS design systems. 
It resolves component behavior, visual expression, and content 
register to the right experience for every user archetype, funnel 
stage, and context — without manual variants, QA overhead, or 
Figma handoff cycles.

## Claim territory
- "Your design system knows how to render. MODE teaches it why."
- The semantic layer above your component library
- Components that know who they're talking to
- Design intelligence for teams that move in code

## Setup, ownership, and extensibility

**Setup:** MODE is designed to be running in under an hour. 
Install the kit, configure your preset once, wire your brand 
tokens. The intelligence layer is operational before your first 
brief.

**Ownership:** MODE lives in your codebase, not a vendor 
dashboard. No runtime dependency, no usage-based billing, no 
lock-in. Fork it, modify it, make it yours. The semantic layer 
is infrastructure you control.

**Extensibility:** The resolution engine is configuration-driven. 
Swap the preset, change the palette map, add components to the 
manifest. MODE adapts to your design system — not the other way 
around. Adapters for shadcn and Radix UI included.

**Features:**
Semantic intent layer across four resolution dimensions: archetype, funnel stage, intent state, context mode
Palette map — visual emphasis resolves per component per funnel stage, no manual assignment
Behavioral tokens — copy density, evidence requirements, CTA rules driven by archetype
IA planner — brief to page architecture in seconds
Multi-variant generation — one brief produces Mover, Validator, Explorer variants in parallel
Component manifest — 11 components fully specified with zone affinity, slot schemas, and usage intent
Token dashboard — editable palette map, accent configuration, preset switching
